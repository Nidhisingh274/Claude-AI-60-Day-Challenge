## Key Learnings

- **ATS systems parse left-to-right, top-to-bottom** — multi-column layouts break parsing completely
- **A flat skills list scores poorly** — categorized skills (Frameworks / APIs / Tools) match JD keywords better
- **Passive language hurts ATS ranking** — strong action verbs (Built, Developed, Implemented) improve both ATS and recruiter readability
- **The top third of a resume is weighted most heavily** — a keyword-rich summary dramatically raises the score
- **Abbreviations cost points** — "ICMR-NICPR" expanded to full name helps ATS match the role correctly
- **Claude can function as a full ATS optimization tool** — from scoring to rewriting to PDF generation in one conversation
